iex Annotations

iex annotations allow you to place notes on an image without actually editing the image. This can be useful for quickly translating image elements without having to edit or upload a new image.

Viewing annotations

Annotations are loaded from posts containing an [iex:a/ANNOTATIONDATA] tag. This tag contains positioning and formatting information for the annotations.

Once the script has detected the tag, the annotations will be applied to the image when it is hovered. Additionally, links to the image (including the tag itself) can be clicked to open the image in a new tab and show the annotations in that tab.

If multiple annotation tags are detected for the same image, the annotations can be toggled through by using the hotkey, which is by default the "A" key.

Opening the editor

The annotation editor can be opened by opening a quick reply form and then doing one of the following:

  • Select a file by using either of the available file selection buttons.
    note that with appchan-x, you may have to select the file twice due to some script incompatabilities
  • Select a file by clicking the URL button and then inputting a URL to an image.
    this is for uploading by URL on 4chan-x and similar extensions
  • Quote a post with an image and the post reference drop-down should auto-select.
    with this method, uploading an image is not necessary

The first two methods apply the annotations to an image uploaded with the post. The second method will apply the annotations to an image that is referenced in the post. This allows raw images to be uploaded and annotations to be added later.

Editor image window

In the annotation editor, there will be an image window and several options for annotation styling.

The image can be scrolled by clicking and dragging with the left/middle mouse buttons, and scaled using the mouse wheel.

Once annotations are added, they can be moved by clicking and dragging; resized by clicking and dragging around the border; and sized-to-fit by double clicking.

Editor styling

The rest of the editor contains styling features, most of which are probably not useful. Includes:

  • Colors - for text, decoration, and background
  • Alignment - vertical and horizontal alignment of the text
  • Font - font family, size, bold, and italic settings for the font
  • Decoration - text decorations in the form of varying shadows

Text formatting

In addition to the styles above, single lines of text can have formatters in them to change how they're displayed. These can be added into the text editors in the editor window to achieve the following effects:

  • \\ - outputs a single backslash character
  • \b - inverts the current font bold setting
  • \i - inverts the current font italic setting
  • \n - outputs a single newline
  • \h - hyphenates a word and inserts a newline
  • \cX.YY: - changes a color
    X. - optional; indicates the color to change (0=text; 1=decoration; 2=background) YY - the index of the color to use; hover over a color in the editor for its color index : - optional; only required to delimit in case the next character is a digit
  • \dX - changes the text decoration
    X - the decoration to change to; hover over the decoration selector to find a decoration index
  • \C - resets all colors to their defaults
  • \D - resets the text decoration to its default
  • \R - resets all text formattings to their defaults

Quick reply editing

Annotations can also be edited directly from the quick reply textarea, though it's not as flexible. This allows for copy-pasting annotations from other sources to quickly continue editing.

The [iex] tag should automatically update once the textarea changes focus.

Settings

The settings includes many options for annotations, including options to disable, modify image URLs or not, transparency of annotations, the hoykey for toggling, and some defaults for the editor.
